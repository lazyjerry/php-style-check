# PHP コーディングスタイルチェック

PHP ファイルが **PER Coding Style 3.0** または **PSR-12** のスタイル規約に準拠しているかを検査する VS Code Copilot スキルです。レポートモード（問題の一覧表示のみ）と修正モード（ファイルを直接修正）をサポートしています。

## 機能

- 現在のファイル、指定した git commit、または git ブランチのスタイル違反をチェック
- **PER Coding Style 3.0**（PHP 8.x の機能に対応）と **PSR-12**（PHP 7.x プロジェクト向け）をサポート
- 2つの実行モード：**レポート**（問題の一覧表示）と **修正**（自動修正）
- 重大度の分類：**Error**（MUST / MUST NOT 規則の違反）と **Warning**（SHOULD / SHOULD NOT 規則の違反）
- 各問題に具体的な行番号を付記した詳細レポート

## トリガーキーワード

ユーザーが以下の内容に言及した場合にこのスキルが起動します：
- PHP コーディングスタイルチェック
- スタイルチェック / フォーマットチェック
- PER style / PSR-12
- コードフォーマット検査

## ワークフロー

### ステップ 1：チェック範囲の確認

| ソース | 取得方法 |
|--------|----------|
| 現在のファイル | 指定パスの PHP ファイルを直接読み取り |
| Git commit | `git show <commit>:<file>` でファイル内容を取得、`git diff-tree` で変更一覧を取得 |
| Git branch | `git diff <base>...<branch>` で差分一覧を取得、作業ディレクトリのファイルを読み取り |

### ステップ 2：スタイル規約の選択

| オプション | 説明 |
|------------|------|
| PER Coding Style 3.0（デフォルト） | PHP 8.x の機能（enum、match、attributes、property hooks）に対応 |
| PSR-12 | PHP 7.x プロジェクト向け |

### ステップ 3：実行モードの選択

| モード | 説明 |
|--------|------|
| レポート（デフォルト） | すべてのスタイル問題を一覧表示し、ファイルは変更しません |
| 修正 | スタイル問題を自動修正し、変更サマリーを出力します |

### ステップ 4〜6：チェックと出力

対応する規約リファレンスファイルを読み取り、セクションごとに優先順位に従ってルールをチェックし、レポートを出力するか修正を実行します。

**チェック優先順位（高→低）：**

1. インデントとスペース
2. 波括弧の位置（制御構造、クラス、メソッド）
3. キーワードと型の大文字・小文字
4. 行の長さと空行
5. 宣言の順序（declare/namespace/use）
6. パラメータリストと戻り値の型フォーマット
7. 演算子のスペース
8. トレーリングカンマ（PER 3.0 のみ）
9. PHP 8.x 機能のフォーマット（PER 3.0 のみ）

### ステップ 7：レポートの保存

完了後、ユーザーはデフォルトパスまたはカスタムパスにレポートを保存できます。

## プロジェクト構成

```
php-style-check/
├── SKILL.md                         # スキル定義
├── references/
│   ├── per-coding-style.md          # PER Coding Style 3.0 リファレンス
│   └── psr-12.md                    # PSR-12 リファレンス
└── templates/
    └── style-report.md              # レポート出力テンプレート
```

## 注意事項

- このスキルは**フォーマットスタイル**（インデント、スペース、波括弧の位置、行の長さなど）に特化しており、コード品質（アーキテクチャ、命名、ロジック）は対象外です
- すべての問題に**具体的な行番号**が付記されており、問題箇所を素早く特定できます
- すべてのチェックに合格した場合でも、レポートには「規約準拠のハイライト」が表示され、良い点が確認できます
- git commit/branch モードでは、`.php` 拡張子のファイルのみがチェック対象です
