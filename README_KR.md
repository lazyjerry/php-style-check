# PHP 코딩 스타일 검사

PHP 파일이 **PER Coding Style 3.0** 또는 **PSR-12** 스타일 규격을 준수하는지 검사하는 VS Code Copilot 스킬입니다. 보고서 모드(문제 목록만 표시)와 수정 모드(파일 직접 수정)를 지원합니다.

## 기능

- 현재 파일, 지정된 git commit 또는 git 브랜치의 스타일 위반 사항 검사
- **PER Coding Style 3.0**(PHP 8.x 기능 지원) 및 **PSR-12**(PHP 7.x 프로젝트용) 지원
- 두 가지 실행 모드: **보고서**(문제 목록 표시) 및 **수정**(자동 수정)
- 심각도 분류: **Error**(MUST / MUST NOT 규칙 위반) 및 **Warning**(SHOULD / SHOULD NOT 규칙 위반)
- 각 문제에 구체적인 줄 번호가 포함된 상세 보고서

## 트리거 키워드

사용자가 다음 내용을 언급하면 이 스킬이 활성화됩니다:
- PHP 코딩 스타일 검사
- 스타일 검사 / 포맷 검사
- PER style / PSR-12
- 코드 포맷팅 검사

## 워크플로

### 1 단계: 검사 범위 확인

| 소스 | 가져오는 방법 |
|------|---------------|
| 현재 파일 | 지정된 경로의 PHP 파일을 직접 읽기 |
| Git commit | `git show <commit>:<file>`로 파일 내용 가져오기, `git diff-tree`로 변경 목록 가져오기 |
| Git branch | `git diff <base>...<branch>`로 차이 목록 가져오기, 작업 디렉터리의 파일 읽기 |

### 2 단계: 스타일 규격 선택

| 옵션 | 설명 |
|------|------|
| PER Coding Style 3.0 (기본값) | PHP 8.x 기능 (enum, match, attributes, property hooks) 지원 |
| PSR-12 | PHP 7.x 프로젝트에 적합 |

### 3 단계: 실행 모드 선택

| 모드 | 설명 |
|------|------|
| 보고서 (기본값) | 모든 스타일 문제를 목록으로 표시하며, 파일을 수정하지 않습니다 |
| 수정 | 스타일 문제를 자동으로 수정하고 변경 요약을 출력합니다 |

### 4~6 단계: 검사 및 출력

해당 규격 참조 파일을 읽어 섹션별로 우선순위에 따라 규칙을 검사하고, 보고서를 출력하거나 수정을 실행합니다.

**검사 우선순위 (높음 → 낮음):**

1. 들여쓰기 및 공백
2. 중괄호 위치 (제어 구조, 클래스, 메서드)
3. 키워드 및 타입 대소문자
4. 줄 길이 및 빈 줄
5. 선언 순서 (declare/namespace/use)
6. 매개변수 목록 및 반환 타입 형식
7. 연산자 공백
8. 후행 쉼표 (PER 3.0 에만 해당)
9. PHP 8.x 기능 형식 (PER 3.0 에만 해당)

### 7 단계: 보고서 저장

완료 후 사용자는 기본 경로 또는 사용자 지정 경로에 보고서를 저장할 수 있습니다.

## 참고 사항

- 이 스킬은 **서식 스타일**(들여쓰기, 공백, 중괄호 위치, 줄 길이 등)에 초점을 맞추며, 코드 품질(아키텍처, 명명, 로직)은 대상이 아닙니다
- 모든 문제에 **구체적인 줄 번호**가 표시되어 문제 위치를 쉽게 파악할 수 있습니다
- 모든 검사를 통과하더라도 보고서에 「규격 준수 하이라이트」가 표시되어 잘한 부분을 확인할 수 있습니다
- git commit/branch 모드에서는 `.php` 확장자 파일만 검사합니다
